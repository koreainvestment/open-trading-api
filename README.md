# Samsung Auto Trader

한국투자증권 Open API를 활용해 삼성전자(`005930`)를 대상으로 작동하는 모의투자 자동매매 시스템입니다.

이 repository는 한국투자증권 Open API 샘플 코드 저장소를 기반으로 하며, 그 안에 `samsung_auto_trader/` 폴더를 추가하여 자동매매 프로그램을 구현한 구조입니다.

이 프로젝트는 실제 투자 수익을 목적으로 하기보다, Open API를 이용한 자동매매 시스템의 기본 흐름을 구현하고 발표하기 위해 제작되었습니다.

---

## 1. 프로젝트 목적

이 프로젝트의 목적은 한국투자증권 Open API를 활용해 자동매매 시스템이 어떤 흐름으로 작동하는지 보여주는 것입니다.

구체적으로는 다음 과정이 하나의 프로그램 안에서 연결되도록 구현했습니다.

```text
현재가 조회
→ 계좌 잔고 조회
→ 매수/매도 조건 판단
→ 주문 요청
→ 주문 후 계좌 재조회
→ 다음 사이클 대기
```

즉, 단순히 API를 한 번 호출하는 코드가 아니라, 실제 자동매매 프로그램처럼 시세 조회, 계좌 조회, 주문 판단, 주문 요청, 결과 확인이 반복적으로 이어지는 구조를 만드는 데 초점을 두었습니다.

## 3. 자동매매 프로젝트 구조

```text
samsung_auto_trader/
├── main.py              # 프로그램 실행 시작점
├── config.py            # API 주소, 계좌번호, 환경변수 설정
├── auth.py              # 접근 토큰 발급 및 캐시 관리
├── api_client.py        # API 요청 공통 처리 및 오류 메시지 처리
├── market_data.py       # 삼성전자 현재가 조회
├── account.py           # 계좌 현금 및 보유 주식 조회
├── orders.py            # 매수/매도 주문 요청
├── trader.py            # 전체 자동매매 흐름 제어
├── logger.py            # 로그 출력 설정
├── token_cache.json     # 접근 토큰 캐시 파일
└── requirements.txt     # 필요한 Python 패키지 목록
```

`token_cache.json`은 실행 과정에서 접근 토큰이 저장될 수 있는 파일입니다.  
보안상 GitHub에 업로드하지 않는 것이 좋습니다.

## 4. 코드 구조 설명

이 프로젝트는 자동매매 과정을 하나의 파일에 모두 작성하지 않고, 기능별로 파일을 분리하여 구성했습니다.

전체 실행 흐름은 `main.py`에서 시작됩니다.  
`main.py`는 설정값을 불러온 뒤 `StockTrader` 객체를 만들고, `trader.run()`을 실행하여 자동매매 루프를 시작합니다.

```text
main.py
→ config.py
→ trader.py
→ auth.py
→ market_data.py
→ account.py
→ orders.py
→ api_client.py
→ logger.py
```

각 파일은 독립적인 역할을 담당하지만, 실제 실행 시에는 `trader.py`가 이 기능들을 순서대로 호출하면서 하나의 자동매매 흐름을 구성합니다.

## 5. 파일별 역할

### `main.py`

프로그램의 실행 시작점입니다.

`config.py`에서 API 설정값을 불러오고, `trader.py`의 `StockTrader`를 실행합니다.  
즉, 실제 매매 로직을 직접 처리하기보다는 전체 프로그램을 시작시키는 역할을 합니다.

---

### `config.py`

API 실행에 필요한 설정값을 관리합니다.

계좌번호, App Key, App Secret은 코드에 직접 적지 않고 환경변수에서 불러오도록 구성했습니다.  
또한 모의투자 API 주소, 토큰 발급 주소, 반복 실행 간격, 거래 가능 시간, API endpoint 정보를 한 곳에서 관리합니다.

필요한 환경변수는 다음과 같습니다.

```text
GH_ACCOUNT      # 계좌번호
GH_APPKEY       # 한국투자증권 Open API App Key
GH_APPSECRET    # 한국투자증권 Open API App Secret
```

이 파일을 따로 분리한 이유는 인증 정보, API 주소, 거래 시간 같은 설정값을 코드 여러 곳에 흩어놓지 않기 위해서입니다.

---

### `auth.py`

한국투자증권 Open API 접근 토큰을 관리합니다.

기존에 발급받은 토큰이 남아 있으면 `token_cache.json`에서 재사용하고, 토큰이 없거나 만료되었으면 새 토큰을 요청합니다.

```text
토큰 캐시 확인
→ 유효한 토큰이면 재사용
→ 만료되었거나 없으면 새 토큰 발급
→ 새 토큰을 token_cache.json에 저장
```

이를 통해 매번 실행할 때마다 불필요하게 새 토큰을 발급받지 않도록 했습니다.

---

### `api_client.py`

한국투자증권 Open API 요청을 공통적으로 처리하는 파일입니다.

다른 파일들이 직접 `requests.get()`이나 `requests.post()`를 반복해서 작성하지 않도록, API 요청을 하나의 `ApiClient` 클래스로 통합했습니다.

이 파일은 다음 역할을 담당합니다.

- API 요청 URL 생성
- 인증 헤더 구성
- GET/POST 요청 처리
- HTTP 오류 처리
- 한국투자증권 API 업무 실패 응답 처리
- 초당 거래건수 초과 오류 처리
- API 요청 재시도 처리
- 응답 JSON 파싱

특히 `EGW00201`처럼 초당 거래건수를 초과한 경우에는 일정 시간 대기한 뒤 재시도하도록 구성했습니다.

즉, `api_client.py`는 이 프로젝트에서 API 통신의 공통 창구 역할을 합니다.

---

### `market_data.py`

삼성전자 현재가를 조회하는 파일입니다.

한국투자증권 시세 조회 API를 호출하고, 응답값에서 현재가를 의미하는 값을 추출합니다.

```text
시세 조회 API 호출
→ API 응답 수신
→ 현재가 데이터 추출
→ 현재가 반환
```

자동매매 판단에서 가장 먼저 필요한 값이 현재가이기 때문에, 이 파일은 매수 가격과 매도 가격을 계산하기 위한 기초 데이터를 제공합니다.

---

### `account.py`

계좌 정보를 조회하는 파일입니다.

계좌의 주문 가능 현금과 삼성전자 보유 수량을 조회합니다.

자동매매에서는 단순히 현재가만 안다고 주문할 수 없습니다.  
매수를 하려면 주문 가능 현금이 충분해야 하고, 매도를 하려면 해당 종목을 보유하고 있어야 합니다.

따라서 `account.py`는 다음 정보를 확인합니다.

```text
주문 가능 현금
보유 주식 수량
조회 대상 종목 코드
```

API 응답 형태가 list 또는 dict로 다르게 들어올 수 있기 때문에, 여러 응답 구조를 고려해 현금과 보유 수량을 파싱하도록 구성했습니다.

이 파일의 결과는 `trader.py`에서 매수 가능 여부와 매도 가능 여부를 판단하는 데 사용됩니다.

---

### `orders.py`

매수/매도 주문을 요청하는 파일입니다.

지정가 주문에 필요한 계좌번호, 종목코드, 주문 수량, 주문 가격을 API 요청 형식에 맞게 구성합니다.

주문 요청 시에는 매수와 매도에 따라 서로 다른 거래 ID를 사용합니다.

```text
매수 주문 → VTTC0802U
매도 주문 → VTTC0801U
```

주문 요청 후에는 API 응답에서 주문번호, 성공 여부, 메시지를 추출합니다.

```text
주문 요청
→ API 응답 수신
→ 주문번호 확인
→ 성공 여부 판단
→ 주문 결과 반환
```

즉, `orders.py`는 실제 주문 요청과 주문 결과 해석을 담당합니다.

---

### `trader.py`

자동매매 시스템의 핵심 제어 파일입니다.

`trader.py`는 다른 파일들의 기능을 순서대로 호출하면서 전체 자동매매 흐름을 만듭니다.

한 번의 매매 사이클은 다음 순서로 진행됩니다.

```text
1. 거래 가능 시간인지 확인
2. 삼성전자 현재가 조회
3. 계좌 현금 및 보유 수량 조회
4. 매수 가격과 매도 가격 계산
5. 매수 가능 여부와 매도 가능 여부 판단
6. 조건이 맞으면 지정가 주문 요청
7. 주문 후 계좌 상태 재조회
8. 체결 여부를 간단히 추정
9. 일정 시간 대기 후 다음 사이클 반복
```

이 파일에서 실제 자동매매 흐름이 연결됩니다.

다만 API 호출, 시세 조회, 계좌 조회, 주문 요청 기능을 모두 직접 처리하지는 않습니다.  
각 기능은 별도 파일에 분리되어 있고, `trader.py`는 이 기능들을 순서대로 호출하는 조정자 역할을 합니다.

---

### `logger.py`

프로그램 실행 로그를 출력하는 파일입니다.

실행 시간, 로그 레벨, 모듈 이름, 메시지가 함께 출력되도록 설정했습니다.

로그를 통해 다음 정보를 확인할 수 있습니다.

- 프로그램 시작 여부
- 현재가 조회 결과
- 계좌 조회 결과
- 매수/매도 판단 결과
- 주문 요청 결과
- 주문 후 계좌 상태
- 오류 발생 여부

자동매매 프로그램은 실행 중 어떤 판단이 이루어졌는지 확인하는 것이 중요하기 때문에, 로그는 실행 흐름을 검증하는 역할을 합니다.

---

### `requirements.txt`

프로그램 실행에 필요한 Python 패키지를 정리한 파일입니다.

이 프로젝트에서는 한국투자증권 Open API에 HTTP 요청을 보내기 위해 `requests` 패키지를 사용합니다.

## 6. 자동매매 흐름

프로그램은 한 사이클마다 다음 순서로 작동합니다.

```text
1. 삼성전자 현재가를 조회한다.
2. 계좌의 주문 가능 현금과 보유 주식 수량을 조회한다.
3. 현재가를 기준으로 매수 가격과 매도 가격을 계산한다.
4. 매수 가능 여부와 매도 가능 여부를 판단한다.
5. 조건이 맞으면 지정가 주문을 요청한다.
6. 주문 후 계좌 상태를 다시 조회한다.
7. 일정 시간 대기한 뒤 다음 사이클을 반복한다.
```

전체 구조를 간단히 나타내면 다음과 같습니다.

```text
[main.py]
    ↓
[config.py] 설정값 로드
    ↓
[trader.py] 자동매매 루프 실행
    ↓
[auth.py] 접근 토큰 확인
    ↓
[market_data.py] 현재가 조회
    ↓
[account.py] 계좌 상태 조회
    ↓
[trader.py] 매수/매도 조건 판단
    ↓
[orders.py] 주문 요청
    ↓
[account.py] 주문 후 계좌 재조회
    ↓
[logger.py] 실행 결과 출력
```

## 7. 매매 로직

이 프로젝트는 복잡한 투자 전략이 아니라, 자동매매 시스템의 구조를 보여주기 위한 단순한 매매 로직을 사용합니다.

```text
매수 가격 = 현재가 - 1,000원
매도 가격 = 현재가 + 1,000원
주문 수량 = 1주
```

매수 조건은 다음과 같습니다.

```text
주문 가능 현금 ≥ 매수 가격 × 주문 수량
```

매도 조건은 다음과 같습니다.

```text
보유 주식 수량 ≥ 주문 수량
```

조건을 만족하면 지정가 주문을 요청합니다.

단, 이 프로젝트는 학습 및 발표 목적이므로 수익성 있는 투자 전략을 구현하는 것보다는 자동매매 시스템이 어떤 구조로 작동하는지를 보여주는 데 초점을 두었습니다.

## 8. 실행 방법

`samsung_auto_trader` 폴더로 이동합니다.

```bash
cd samsung_auto_trader
```

필요한 패키지를 설치합니다.

```bash
pip install -r requirements.txt
```

프로그램을 실행합니다.

```bash
python main.py
```

실행 중인 프로그램을 멈추려면 터미널에서 `Ctrl + C`를 누릅니다.

로그를 통해 현재가 조회, 계좌 조회, 주문 판단, 주문 요청 결과를 확인할 수 있습니다.

## 11. 환경변수 설정

이 프로젝트는 API Key와 App Secret을 코드에 직접 작성하지 않고 환경변수에서 불러오도록 구성했습니다.

필요한 환경변수는 다음과 같습니다.

```text
GH_ACCOUNT
GH_APPKEY
GH_APPSECRET
```

예시는 다음과 같습니다.

```bash
export GH_ACCOUNT="계좌번호"
export GH_APPKEY="앱키"
export GH_APPSECRET="앱시크릿"
```

GitHub Codespaces를 사용하는 경우에는 Codespaces Secrets에 위 값을 등록해서 사용할 수 있습니다.

## 12. 자주 볼 수 있는 메시지

### `모의투자 장종료 입니다.`

주문 요청은 정상적으로 전송되었지만, 모의투자 서버에서 주문 가능 시간이 아니라고 판단한 경우입니다.

이는 코드 오류가 아니라 API 업무상 거절입니다.

### `EGW00201`

초당 거래건수를 초과했다는 의미입니다.

이 프로젝트에서는 API 호출 사이에 대기 시간을 두고, 해당 오류가 발생하면 일정 시간 후 재시도하도록 구성했습니다.

## 13. 주의사항

- 본 프로젝트는 학습 및 발표용 모의투자 시스템입니다.
- 실제 투자 수익을 보장하지 않습니다.
- 지정가 주문 방식이므로 주문이 제출되어도 바로 체결되지 않을 수 있습니다.
- 현재 코드는 미체결 주문을 다음 사이클까지 추적하지 않습니다.
- 주문 후 계좌 상태를 다시 조회하여 체결 여부를 간단히 추정합니다.
- API Key와 App Secret은 코드에 직접 작성하지 않고 환경변수 또는 GitHub Codespaces Secrets로 관리해야 합니다.
- `token_cache.json`은 접근 토큰이 저장될 수 있으므로 GitHub에 업로드하지 않는 것이 좋습니다.
- 한국투자증권 Open API의 모의투자 서버 상태나 거래 가능 시간에 따라 주문이 거절될 수 있습니다.

## 14. 프로젝트 의의

이 프로젝트는 단순한 매매 전략보다 자동매매 시스템의 기본 구조를 이해하는 데 초점을 둡니다.

한국투자증권 Open API를 활용해 다음 기능들이 어떻게 연결되는지 확인할 수 있습니다.

```text
인증
→ 시세 조회
→ 계좌 조회
→ 주문 판단
→ 주문 요청
→ 결과 확인
```

또한 기능별로 파일을 분리하여 자동매매 프로그램을 보다 이해하기 쉬운 구조로 구성했습니다.

특히 `trader.py`가 전체 흐름을 제어하고, `api_client.py`가 API 요청을 공통 처리하며, `market_data.py`, `account.py`, `orders.py`가 각각 시세 조회, 계좌 조회, 주문 요청을 담당하도록 나눈 점이 이 프로젝트의 핵심 구조입니다.
